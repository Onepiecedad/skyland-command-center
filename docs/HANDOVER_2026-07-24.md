## Hur dispatchen fungerar nu (pull-modellen)

1. **SCC (Render):** env `OPENCLAW_DISPATCH_MODE=pull`. Dispatchern pushar inte — den KÖAR
   claw-körningar (`task_runs.worker_id='pull:queued'`, task `in_progress`).
2. **Macen:** `openclaw-config/scripts/scc_poller.py` kör via launchd (`com.skyland.scc-poller`,
   KeepAlive). Hämtar köade körningar via `GET /api/v1/claw/pending?worker=…` (Bearer `SCC_API_TOKEN`),
   claimar atomiskt (`pull:queued → pull:claimed`), och avfyrar agenten på gatewayns `/hooks/agent`.
3. **Agenten** kör lokalt och rapporterar tillbaka via `scc-callback`-skillen →
   `/api/v1/claw/task-result`. Reapern fångar körningar som aldrig rapporterar (timeout → failed).

Push-läget (default `push`) finns kvar för lokal körning / ev. framtida tunnel.

## Grindarna som lagades (felsökningskedjan)

1. **Deploy inte utrullad** — nya commits låg i main men Render körde gammal version. Manual Deploy.
2. **Monterings-bugg (ROTORSAK):** `server.ts` monterade dispatch-routern på `/api/v1/dispatch`, men
   frontend-dispatchknappen, `scc-callback`, `taskService` och legacy `index.ts` förväntade `/api/v1`.
   En entrypoint-refaktor hade tyst brutit dispatch + callbacks + dispatch-knappen. **Fix: montera om
   till `/api/v1`** (server.ts, en rad) → allt konsekvent.
3. **Callback-sökvägar** återställda till `/api/v1` (efter monteringsfixen).
4. **Agent-registrering:** `produce-package` + `prospect-finder` fanns som mappar men saknades i
   `openclaw.json` `agents.list` → gateway gav 400. Registrerade båda (id/model/workspace), kopierade
   workspaces till `~/clawd/_agents/`, `deploy_openclaw_config.sh`, gateway-omstart.
5. **sessionKey:** `/hooks/agent` kräver `sessionKey` som börjar med `hook:` — pollern skickade `scc:`
   → 400. Bytte till `hook:scc-…`. (Detta, inte agenten, var den faktiska 400:an.)
6. **py3.9-kompat:** launchd kör system-python 3.9 (interaktivt var 3.14) → `dict | None`-annoteringar
   kraschade. `from __future__ import annotations`.

## Viktigt att veta

- **Två Alexar:** verktygen `produce_package` / `find_prospects` finns BARA på **server-Alex** — den
  flytande docken (`⌘J`, `/api/v1/chat/chat`, "server-läge · full CRM-åtkomst"). Den stora Alex-vyn är
  **gateway-Alex** (WebSocket mot gatewayn) och frilansar med egna skript — den skapar INTE tasks.
- **Gatewayn** är en launchd-tjänst `ai.openclaw.gateway` (watchdog: `ai.openclaw.gateway.watchdog`).
  Omstart efter config-ändring: `launchctl kickstart -k gui/$(id -u)/ai.openclaw.gateway`.
- **Pollern** startas/stoppas via `com.skyland.scc-poller` (plist i `openclaw-config/launchd/`).
  Omstart efter kod-ändring: `launchctl kickstart -k gui/$(id -u)/com.skyland.scc-poller`.
- **openclaw.json** synkas repo→runtime med `deploy_openclaw_config.sh` (backup + validering); den
  laddas INTE om automatiskt — gatewayn måste startas om.
